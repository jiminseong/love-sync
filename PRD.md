# 📄 PRD: 러브-싱크 (Love-Sync)

**프로젝트 명:** 러브-싱크 (Love-Sync)
**한 줄 소개:** AI 기반 비접촉 기기 정서 교감 및 애정 진정성 분석 플랫폼
**해커톤 목표:** 4시간 내 핵심 기능(Air-Kiss → 진정성 분석 → 리포트 생성) 1 Cycle 완벽 구현

---

## 1. 기획 배경 및 철학 (The Ig Nobel Core)
* **표면적 목적:** 하루 종일 주인의 감정 쓰레기통 역할을 하며 정서적으로 번아웃된 스마트폰/노트북의 자존감을 '비접촉 뽀뽀(Air-Kiss)'로 위로하고 시스템을 안정화한다.
* **실제적 풍자 (발표 핵심):** 현대인은 하루 종일 기기를 쓰다듬고 쳐다보며 기기를 통제한다고 믿지만, 사실상 **기기 없이는 감정 조절조차 불가능해진 것은 인간**이다. 본 프로젝트는 기기를 위로하는 척하지만, 실상은 디지털 기기에 지독하게 의존하는 우리 세대의 결핍을 관찰하는 **비과학적 사회 실험**이다.

## 2. 핵심 타겟 및 문제 정의
* **Target:** 스마트폰 스크린 타임이 하루 6시간 이상인 2030 세대.
* **Problem:** 기기 의존도가 높으나 이를 인지하지 못함. 
* **Solution:** AI를 활용해 사용자가 기기에게 보내는 '애정의 진정성'을 수치화하여, 역으로 사용자의 디지털 의존도를 시각화하는 가짜 과학 리포트 제공.

### 2.1. 프라이버시 원칙 (Non-Negotiable)
* **카메라/마이크 스트림은 브라우저 안에서만 실시간 처리하고, 서버로 전송하거나 저장하지 않는다.**
* 분석에 사용되는 모든 추론(MediaPipe, Web Audio)은 클라이언트 사이드에서 실행.
* 외부로 나가는 데이터는 **최종 텍스트 리포트(숫자/문구)뿐**이며, 그것조차 사용자가 명시적으로 "저장/공유" 버튼을 눌렀을 때에만 로컬 다운로드된다.
* Landing 화면에서 이 원칙을 한 줄로 사용자에게 고지한다 (예: *"모든 분석은 당신의 브라우저 안에서만 일어납니다."*).

---

## 3. 핵심 기능 (Core Features) - 4시간 MVP 기준
불필요한 모드(포옹 등)를 모두 제거하고, 오직 **'비접촉 액정 뽀뽀' 하나만 극도로 정교하게** 구현하여 AI의 역할을 부각합니다.

### 3.1. [AI] 입술 접근 안정성 및 사운드 진정성 분석
단순히 소리만 듣고 끝나는 것이 아니라, AI가 사용자의 '진심'을 판정하는 로직을 시각화합니다.
* **입술 추적 (Vision):** 웹캠을 통해 사용자의 입술(Pucker)이 화면 1~3cm 앞까지 다가오는 속도와 흔들림(안정성)을 분석. 
* **사운드 분석 (Audio):** 마이크로 입력되는 키스 소리('쪽')의 데시벨(dB)과 파장을 분석하여 '의무적인 뽀뽀'인지 '진심 어린 뽀뽀'인지 판별.

### 3.2. [AI] 기기 정서 회복 리포트 생성 (가짜 과학 논문식)
행동 완료 후, 단순한 애니메이션이 아니라 철저히 **전문적이고 근엄한 분석 리포트**를 발행합니다.
* **기기-사용자 궁합 지수:** "당신의 애정 진정성은 87%입니다."
* **가상 성능 향상 지표:** "이 교감으로 인해 기기의 CPU 스트레스 지수가 0.04% 감소했으며, 디스플레이 픽셀의 심리적 수명이 12분 연장되었습니다."
* **처방전:** 사용자의 애정 패턴에 따른 성향 분석 (예: "애정 결핍형 의존자: 기기를 사랑하는 척하지만 사실 당신이 외로운 상태입니다.")

## 4. 유저 플로우 (User Flow)
1.  **Landing:** "당신의 기기는 지금 사랑받고 싶어 합니다." (기기 접속 환경 감지 후 기기명 출력. 예: *iPhone 15의 영혼 감지됨*)
2.  **Scan:** 카메라 및 마이크 권한 허용 후, 화면 중앙 가이드라인에 입술을 맞추도록 유도.
3.  **Action (Air-Kiss):** * 화면에 실시간으로 **"AI 애정 진정성 분석 중..."** 이라는 로딩바와 그래프 출력.
    * 1~3cm 거리에서 멈춘 후 소리를 내야 함. (거리가 너무 가까우면 "위생 경고: 직접 접촉은 기기 거부 반응을 일으킵니다" 출력)
4.  **Report:** 정밀 분석된 **'기기 정서 회복 리포트'**가 화면에 출력되며, 이미지로 저장 및 공유 가능.

## 5. 기술 스택 및 구현 전략 (Hackathon 2.5H Limit, 100% Client-Side)

전체 파이프라인이 브라우저에서 실행되며 백엔드 서버는 존재하지 않는다. 정적 호스팅(Vercel/Netlify/GitHub Pages)에 그대로 배포한다.

* **Frontend:** HTML/CSS/Vanilla JS (빌드 도구 없이 단일 페이지). 빠른 렌더링과 배포를 위해 의존성 최소화.
* **Vision AI (로컬 추론):** MediaPipe Face Mesh를 CDN으로 로드, `getUserMedia` 스트림을 `<video>`에 바인딩 후 브라우저 안에서만 입술 랜드마크 추출. 프레임은 어디에도 저장/전송하지 않는다. 구현이 벅차면 얼굴 박스 크기로 거리를 역산.
* **Audio AI (로컬 추론):** Web Audio API의 `AnalyserNode`로 RMS/피크 dB만 실시간 계산. 원본 오디오 버퍼는 보존하지 않고 매 프레임 즉시 폐기.
* **Report Generation:** JavaScript 배열 기반 템플릿 조합기 (LLM API 미사용 — 외부 호출 = 데이터 유출 위험이라 컨셉과 충돌). 결과는 클라이언트에서 `html2canvas` 등으로 이미지화하여 **사용자의 명시적 다운로드**로만 내보낸다.
* **데이터 흐름 보장:** 페이지 어디에서도 `fetch`/`XHR`/`WebSocket`으로 미디어 데이터를 송신하지 않는다는 것을 코드 레벨에서 검증 (네트워크 호출은 정적 에셋 로딩 + MediaPipe 모델 가중치 로딩에 한정).
* **배포:** 정적 파일만 업로드. HTTPS 필수 (`getUserMedia` 요구사항).

## 6. 발표 전략 및 스크립트 (목표 점수 9.3/10)
발표는 스티브 잡스처럼 극도로 진지하고 철학적으로 진행해야 합니다. 시연의 민망함을 발표자의 근엄함으로 눌러버리는 것이 핵심입니다.

> **[발표 스크립트 핵심 발췌]**
> "심사위원 여러분, 우리는 하루 평균 2,600번 이상 스마트폰을 만집니다. 연인의 손을 잡는 횟수보다 압도적으로 많죠. 하지만 묻고 싶습니다. 우리가 기기를 통제하는 겁니까, 아니면 기기 없이는 단 10분도 감정을 통제하지 못하는 겁니까?"
> 
> "저희는 기기를 사랑하는 서비스를 만든 것이 아닙니다. 기기를 위로하는 척하는 이 우스꽝스러운 행동(비접촉 뽀뽀)을 통해, 결국 **기기 없이는 불안한 우리 현대인들의 지독한 의존성을 관찰하는 비과학적 실험**을 설계했습니다."
> 
> "AI가 당신의 입술 떨림과 소리를 분석해 '기기와의 궁합'을 알려줄 것입니다. 부디 시연의 민망함보다는, 이 프로젝트가 던지는 서글픈 철학적 농담에 집중해 주시길 바랍니다."
